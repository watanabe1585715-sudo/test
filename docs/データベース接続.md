# データベース接続情報（PostgreSQL）

このドキュメントは、ローカル開発環境で PostgreSQL に接続するための手順書です。  
接続値は Docker Compose の既定値に基づいています（DB のみ起動時も同一）。

## 接続パラメータ一覧

| 項目 | 値 |
|------|-----|
| 種別 | **PostgreSQL** |
| ホスト | `127.0.0.1` または `localhost` |
| ポート | `5432` |
| データベース名（DB 名） | `recruit` |
| ユーザー名 | `recruit` |
| パスワード | `recruit` |
| SSL | 無効（ローカル開発向け） |

**接続 URL（アプリ・ツール共通の例）**

```text
postgres://recruit:recruit@127.0.0.1:5432/recruit?sslmode=disable
```

`.env` の `DATABASE_URL` にそのまま使えます。

## A5:SQL Mk-2 で接続する手順

[A5:SQL Mk-2](https://a5m2.com/)（無料の DB クライアント）で、テーブルやデータを確認する場合の例です。

1. **Docker で DB を起動**しておく（例: `make db` または `docker compose up -d db`）。
2. A5:SQL Mk-2 を起動し、**データベース** → **追加**（または接続ウィザード）。
3. **データベースの種類**で **PostgreSQL** を選ぶ。
4. 次の内容を入力します。
   - **ホスト名**: `127.0.0.1`
   - **ポート番号**: `5432`
   - **データベース名**: `recruit`
   - **ユーザー名**: `recruit`
   - **パスワード**: `recruit`
   - SSL 設定がある場合は **無効**（`disable` 相当）にする（表記は環境により異なります）。
5. **接続テスト** → 成功したら保存して接続。

WSL2 上の Linux と、Windows 側の A5 を組み合わせる場合でも、PostgreSQL が WSL のポートに公開されていれば **ホストは `localhost`** のままで動作することが多いです。  
接続できない場合は、Docker のポートマッピング（`5432:5432`）とファイアウォール設定を確認してください。

## よく使うテーブル（確認用）

| テーブル | 内容 |
|----------|------|
| `customers` | 顧客・契約 tier・契約期間・`approval_status`（公開求人は `approved` のみ） |
| `job_postings` | 案件・掲載期間・`publication_status` |
| `applications` | 応募 |
| `job_admin_users` | 案件管理サイト用ログイン |
| `customer_admin_users` | 顧客管理サイト用ログイン（`registration_status`: 承認待ち / 承認済 / 却下） |
| `customer_events` | 顧客の打ち合わせ・利用開始・特記・リスク関連などの履歴 |
| `job_seeker_accounts` | 求人サイト求職者ログイン |
| `job_seeker_profiles` | 求職者プロフィール（1 アカウント 1 行） |
| `invoices` | 請求 |
| `prospects` | 見込み顧客 |
| `schema_migrations` | どの SQL まで流したか（自動） |

スキーマ追加例は `007_jobseeker_customer_events.sql`（求職者・顧客承認・顧客イベント）です。

初期データは `internal/migrate/sql/002_seed.sql` と `003_seed_demo.sql` にあります。  
**新しいシードを取り込みたい**場合は、開発用ボリュームを削除して DB を再作成する方法が簡単です。

```bash
docker compose down -v
docker compose up -d db
# その後 API を起動するとマイグレーションが再実行される
```
